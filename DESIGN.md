# Lacuna Design

Lacuna is designed around **the gap**: surfaces are revealed from seams,
attachment edges remain structurally honest, absence carries visual weight,
and the active Omarchy theme owns hue while Lacuna owns form.

The authoritative design-system documentation is
[`docs/lacuna-design-system/README.md`](docs/lacuna-design-system/README.md).
Its specifications are implementation contracts, not optional inspiration:

- [Philosophy and principles](docs/lacuna-design-system/00-philosophy.md)
- [Color roles and theme ownership](docs/lacuna-design-system/01-color.md)
- [Geometry, seams, and molding connectors](docs/lacuna-design-system/02-geometry.md)
- [Reveal motion and choreography](docs/lacuna-design-system/03-motion.md)
- [Typography and type scale](docs/lacuna-design-system/04-typography.md)
- [Components and interaction state laws](docs/lacuna-design-system/05-components.md)
- [Design-system migration status](docs/lacuna-design-system/06-roadmap.md)

Before changing a visible Lacuna surface, read the relevant specifications and
validate geometry, state, and motion behavior—not only source-string contracts.
